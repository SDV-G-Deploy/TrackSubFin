# TrackSubFin

Лёгкий MVP веб‑приложения для трекинга подписок.

- Локальный кэш: `localStorage`
- Облачный MVP sync: **GitHub Gist API** (по Archive Code + GitHub PAT)

## Демо (GitHub Pages)

**https://sdv-g-deploy.github.io/TrackSubFin/**

## Что умеет

- Добавление/удаление подписок с полями:
  - Название продукта
  - Цена
  - Периодичность: `monthly` / `yearly` / `custom-days`
  - Дата следующего списания
  - Карта: название + последние 4 цифры
  - Опционально дата окончания
- Визуализация:
  - Список подписок
  - Горизонтальная timeline по ближайшим списаниям
- Цветовые статусы по ближайшему списанию:
  - `<= 3 дней` — красный + заметное свечение
  - `4–7 дней` — amber
  - `> 7 дней` — нейтрально
- Показ «сколько осталось» до списания и (если есть) окончания
- Итоговая ежемесячная сумма:
  - `yearly -> monthly = price / 12`
  - `custom-days -> monthly = price * (30 / days)`

## Cloud sync (MVP) через GitHub Gist

### Как включить

1. Нажмите **«Синхронизация»**.
2. Введите:
   - **Archive Code** (6–24, латиница/цифры/дефис)
   - **GitHub Token (PAT)** со scope `gist`
3. Нажмите **«Подключить»**.

Приложение ищет (или создаёт) gist:

- `description`: `TrackSubFin Sync <Archive Code>`
- файл: `tracksubfin-data.json`

### Поведение

- Если gist уже существует, его данные загружаются в приложение при подключении (облако = source of truth на момент подключения).
- После изменений подписок срабатывает автосохранение в gist (debounce ~900ms).
- Есть кнопка **«Синхронизировать сейчас»**.
- Есть кнопка **«Отключить»** (удаляет локально token/gist id, локальные подписки остаются).

## Как создать GitHub PAT для `gist`

### Вариант 1: Classic token

1. GitHub → **Settings** → **Developer settings** → **Personal access tokens** → **Tokens (classic)**.
2. **Generate new token (classic)**.
3. Отметьте scope **`gist`**.
4. Скопируйте токен и вставьте в TrackSubFin.

### Вариант 2: Fine-grained token

GitHub постепенно продвигает fine-grained PAT, но доступ к Gist может быть ограничен/зависеть от текущей политики GitHub. Для MVP надёжнее использовать classic PAT с `gist`.

## Подключение 2 устройств

1. На устройстве A подключите sync с Archive Code + PAT.
2. На устройстве B введите **тот же Archive Code** и PAT (или другой PAT с доступом к gist владельца).
3. После подключения устройство B подтянет данные из того же gist.
4. Дальше изменения на каждом устройстве отправляются в тот же gist.

## Ограничения и риски MVP

- PAT и sync-конфиг хранятся в `localStorage` браузера (удобно, но не безопасно как vault/OS keychain).
- Нет полноценного conflict resolution: при подключении облако приоритетно, при ручной/авто-синхронизации отправляется текущее локальное состояние.
- Поиск gist сделан по `description`; при большом числе gist возможно потребуется пагинация/оптимизация.
- Можно упереться в rate-limit GitHub API.

## Локальный запуск

Откройте `index.html` в браузере.
